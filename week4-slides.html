The Mindset
Wrong and fast
> right and perfect.
The Reality
  • Your COS will break
  • A prompt that worked last week will misfire
  • An API will change, a permission will lapse
  • That's normal – it's a living system, not a product
The Habit
  • Fix fast – don't avoid the failure
  • Ship the messy version, then iterate
  • Every bug is a chance to make it sharper
  • Maintenance is the work now
The Three Modes
How to Extend Your COS
Mode 1 · New Skill

Describe what you want.

Tell your COS the outcome – the trigger, the inputs, the output. Let it build the command end-to-end. You don't spec the code, you spec the result.

Mode 2 · Debug

Share the error. Say fix this.

Paste the error, paste the context, name what you were trying to do. Your COS will diagnose, patch, and test. No guessing required from you.

Mode 3 · Upgrade

"Here's what I want this to do better."

Point at an existing command. Tell it what's missing, what's noisy, what should change. Let it refactor – then run the new version live.

The Superpower
Claude Code – built for people
who don't write code.

You describe outcomes. It writes, tests, and deploys the system. The workflow is the same every time:
